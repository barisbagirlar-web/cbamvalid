# Production User Acceptance Flow

```text
register / sign in
→ create draft dossier
→ complete eight preparation sections
→ upload and link evidence
→ resolve readiness blockers
→ select the dossier on the $150 purchase page
→ complete controlled Paddle checkout
→ verify webhook, order, ledger and five entitlements
→ seal one report version
→ verify remaining versions change 5 → 4
→ download 23-component ZIP
→ verify manifest hashes
→ re-download and verify remaining versions stay 4
```

The acceptance run must retain route statuses, browser trace/video, transaction/event/order/ledger/entitlement/release IDs, package artifact, manifest verification output and runtime-log correlation identifiers.
